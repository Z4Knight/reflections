何时你会想使用远程版本库，而不是将所有工作保存在本地？
答 ：与别人进行协同工作或者自己想要在多台计算机之间进行工作。

为何你希望始终手动地拉取更改，而不是让 Git 自
动与你的远程版本库保持同步？
答：因为有时候，可能修改的内容并不是自己满意的情况。

说明Fork、克隆和分支之间的不同。何时使用某种功能而不是另一种？
答：Fork是用于 GitHub 上相同的仓库之间，克隆是用于 GitHub 
与本地计算机，或者本地计算机的不同位置上，不同或相同仓库之间。分支是
用于本地计算机与 git 之间。

拥有远程分支的上次已知状态的副本（存储在本地）有何好处？
答：方便于解决本地分支与远程分支的冲突。
